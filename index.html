The Problem

Headless browsers are a dead end.

Every browser automation tool makes you start from scratch. Fresh session. No cookies. No logins. No 2FA tokens.

Browser Control MCP controls the browser you already have open. The one where you're already logged into GitHub, AWS, Jira, and everything else. The AI works with your real session — not a sterile sandbox.

Setup

Three steps. Thirty seconds.

1

Install the Chrome extension

One click from the Chrome Web Store.

2

Add the MCP server to Claude Code

One line in your config:

claude mcp add browser-control -- npx browser-control-mcp
3

Use your browser from the terminal

Claude Code can now navigate, click, fill, screenshot, and automate your browser.

FAQ

Common questions.

Is my data safe?

All automation happens over localhost. No analytics, no telemetry. Only external call is license validation to LemonSqueezy.

Sites that block bots?

Works. Your real browser, real cookies, real session. No headless fingerprint.

2FA-protected sites?

That's the point. You're already authenticated. No re-auth needed.

React / SPA sites?

Yes. Native value setters + proper event dispatch. React, Vue, Angular all work.

What do I need?

Chrome, Node.js, and Claude Code. That's it.
